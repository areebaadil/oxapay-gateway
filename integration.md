## Overview

This payment gateway allows merchants to accept cryptocurrency payments (USDT) from their customers. The integration uses a simple REST API with API key authentication.

### Key Concepts

- **Hosted Payment Page**: We provide a fully hosted, branded payment page. Your app redirects users there — no need to build payment UI.
- **API Key Authentication**: Every request must include your merchant API key.
- **Webhook Callbacks**: We notify your server when payment status changes.
- **Two Integration Modes**:
  - **Hosted Mode** (Recommended): Provide `success_url` and `failure_url` — customer is redirected to our hosted page.
  - **Direct Mode**: Omit redirect URLs — you receive raw payment details (address, QR code) to build a custom UI.

### Step 2: Create a Payment (Hosted Mode — Recommended)

When a customer is ready to pay, make a server-side API call:

```bash
curl -X POST \
  https://hhioqglejbjcqcmqqsnj.supabase.co/functions/v1/merchant-api/payments \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 50.00,
    "currency": "USD",
    "pay_currency": "USDT",
    "order_id": "ORDER-12345",
    "description": "Purchase from MyStore",
    "success_url": "https://yoursite.com/payment/success",
    "failure_url": "https://yoursite.com/payment/failed",
    "callback_url": "https://yoursite.com/api/webhook/payment"
  }'
```

**Response:**

```json
{
  "success": true,
  "data": {
    "payment_id": "uuid-of-payment",
    "deposit_intent_id": "uuid-of-intent",
    "hosted_payment_url": "https://paygate1.lovable.app/deposit/uuid-of-payment",
    "coin": "USDT",
    "expected_amount": 50.00,
    "expires_at": "2025-01-01T01:00:00.000Z",
    "merchant_name": "Your Merchant Name"
  }
}
```

### Step 3: Redirect the Customer

Redirect your customer to the `hosted_payment_url` from the response:

```javascript
// JavaScript / Node.js example
app.post('/checkout', async (req, res) => {
  const response = await fetch(API_BASE + '/payments', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.MERCHANT_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      amount: req.body.amount,
      order_id: req.body.orderId,
      success_url: 'https://yoursite.com/payment/success',
      failure_url: 'https://yoursite.com/payment/failed',
      callback_url: 'https://yoursite.com/api/webhooks/payment',
    }),
  });

  const { data } = await response.json();
  
  // Redirect user to the hosted payment page
  res.redirect(data.hosted_payment_url);
});
```

### Step 4: Handle the Webhook

When the payment status changes, we send a POST request to your `callback_url`:

```javascript
// Node.js / Express webhook handler
app.post('/api/webhooks/payment', express.json(), async (req, res) => {
  const { status, trackId, amount, orderId } = req.body;

  switch (status) {
    case 'CONFIRMED':
      // Payment successful — fulfill the order
      await fulfillOrder(orderId);
      break;
    case 'FAILED':
      // Payment failed
      await cancelOrder(orderId);
      break;
    case 'EXPIRED':
      // Payment expired — customer didn't pay in time
      await expireOrder(orderId);
      break;
  }

  res.status(200).json({ received: true });
});
```

| Event | Status | Description |

## Error Handling

All error responses follow this format:

```json
{
  "error": "Description of the error"
}
```

| Status Code | Meaning |
|-------------|---------|
| 400 | Bad request — missing or invalid parameters |
| 401 | Unauthorized — invalid or missing API key |
| 404 | Not found — resource doesn't exist |
| 405 | Method not allowed |
| 500 | Internal server error |

## FAQ

### Where is the payment page hosted?

The payment page is **hosted on our platform** (not on your server). When you create a payment with `success_url` and `failure_url`, you receive a `hosted_payment_url`. Simply redirect your customer to that URL. We handle the entire payment experience — QR code display, real-time status, countdown timer, and automatic redirection back to your site upon completion.

### Do I need to build a payment UI?

**No.** If you use the Hosted Mode (recommended), we provide the complete payment interface. Your customer sees a professional, branded payment page. You only need to handle the redirect and webhook.

### Can I customize the payment page?

The hosted payment page displays your merchant name automatically. Further customization options may be available — contact your administrator.

### What currencies are supported?

Currently, **USDT (Tether)** is supported for payments. Amounts are specified in USD.

### How long does a payment session last?

Payment sessions expire after **1 hour**. If the customer doesn't pay within that window, the payment is marked as `EXPIRED`.

### Is my API key safe?

Your API key should **only** be used in server-side code. Never expose it in frontend JavaScript, mobile apps, or any client-side code. Treat it like a password.
