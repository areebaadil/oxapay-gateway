Deposit Workflow Steps

1

Create Deposit Intent

Call POST /create-deposit-intent with merchant_id, user_reference, coin, and expected_amount. Returns a hosted deposit page URL.

2

Redirect User

Redirect the user to the hosted_deposit_page_url returned in step 1. User sees the deposit page with QR code and address.

3

User Makes Payment

User scans QR code or copies the wallet address and sends crypto from their wallet.

4

OxaPay Processes

OxaPay detects the incoming transaction and sends a webhook to /oxapay-webhook.

5

Transaction Confirmed

System updates transaction status to CONFIRMED, creates ledger entries, and sends webhook to merchant's callback URL.

Webhook Event Types

W

payment.waiting

Initial state when payment is created. Waiting for user to send crypto to the address.

C

payment.confirming

Transaction detected on blockchain, waiting for confirmations. Transaction status: PENDING.

✓

payment.confirmed

Payment fully confirmed. Transaction status: CONFIRMED. Ledger entries created. Merchant webhook sent.

X

payment.expired

Payment window expired without receiving funds. Transaction status: EXPIRED.
